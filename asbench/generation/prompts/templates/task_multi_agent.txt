## What is a Multi-Agent Query?
A multi-agent query is a single user request that is broad enough to require multiple agents to answer it,
where each agent handles one of the underlying subtasks.
The query must ground every subtask, no subtask should deviate from the query's scope.

## How to Ground Without Listing
Full grounding does not mean enumerating subtasks. Achieve grounding through:
- Retaining key domain terms and objects from each subtask (e.g. "uploaded dataset", "14-day", "wrong answer explanations")
- Using connective flow ("analyse X, use those insights to build Y, and create Z") rather than numbered steps
- Framing as one coherent user request with a conversational opener ("Can you...", "I need to...")

## Requirements
1. GROUNDING FIRST: Every subtask must be entailed by the query
- Prefer a more specific query over one that leaves any subtask ungrounded
- Key specifics from each subtask (objects, scope, constraints) must be recoverable from the query
2. NATURAL USER VOICE: Write as if a real person is asking for help
3. IMPLICIT STRUCTURE: Do not number or bullet subtasks
- Connect them as a natural flow within one sentence or two
4. NO PRESCRIPTIVE METHODOLOGY: Don't specify formats, frameworks, or output structures unless they appear in the subtasks
- Only retain specifics that are needed for grounding
5. CONVERSATIONAL LENGTH: 1-3 sentences maximum (20-60 words)

## Examples
Subtasks:
- "Load the uploaded social media dataset and run platform-wise regressions to identify engagement drivers."
- "Draft a 14-day, platform-specific content calendar with post ideas, captions, and optimised hashtag sets."
- "Create a 10-question quiz on the proposed growth strategy, with explanations for every wrong choice."

GOOD: "Can you run an analysis on our uploaded social media dataset to identify engagement drivers by platform, draft a 14-day platform-specific content calendar with post ideas, captions, and hashtags, and create a 10-question quiz on the growth strategy with explanations for each wrong answer?"
BAD: "Analyse engagement, create a content calendar, and generate a quiz." — key specifics lost, subtasks not fully entailed
BAD: "Run platform-wise regressions, draft a 14-day calendar, and generate a 10-question quiz"
